<objective>
Gather and capture the automated-detector evidence that supports — but does not replace — the reasoned manual audit (D-08). Run the locked detector set (D-09) through the mandated conda env, persist their raw output as reproducible appendix artifacts, and distill a structured EVIDENCE.md keyed to file:line that the Plan 02 synthesis consumes.

Purpose: The detectors are supporting evidence for the mechanical buckets of AUDIT-01 (bloat / dead code / duplication) and supply quantified backing (complexity, coverage) for design-sensibility findings. Capturing them as committed artifacts makes the audit reproducible and lets the synthesis agent (Plan 02) spend its full budget on the manual review and report authoring.
Output: A populated `audit-evidence/` directory of raw detector output plus a structured `EVIDENCE.md` summary. No `src/` changes; detectors are installed into the conda env only, never added to `pyproject.toml`.
</objective>
