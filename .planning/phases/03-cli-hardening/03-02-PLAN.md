<objective>
Harden the package CLI surface (`cli.py`) so every command validates arguments and exits
with a documented, predictable code. This plan lands the CLI-level routed findings:

- **D-09** — promote `rescore` to its own subcommand so the four rescore-only flags
  (`--reuse-parser-options`, `--use-prediction-cache`, `--pass-id`, the rescore meaning of
  `--max-trials`) live on it and argparse structurally rejects them on `run`.
- **D-08 / F-06** — a minimal POSIX `0` / `1` / `2` exit-code taxonomy; base the
  `rescore`/`analyze` non-zero code on GENUINE errors (`parser_misses` for rescore, genuine
  read/eval failures for analyze), NOT on the presence of pre-existing `skipped_failed`
  trials.
- **D-11 / D-10** — wrap load+dispatch in `main()` in a clean-error handler that catches
  `FileNotFoundError` / `ValueError` (config loaders) and `ToolPreflightError` (the Plan-01
  preflight), prints `error: <msg>` to stderr with no traceback, and returns 1; keep
  argparse's native exit 2 strictly for usage. Add an optional `--traceback` escape hatch.
- **D-04** — add `--timeout` to `run` and wire it to override the tool's
  `execution.timeout_seconds` using `is not None` precedence (so `--timeout 0` is not
  silently dropped).
- **Discretion cleanups** — `_cmd_dashboard` returns 1 (not 2) when streamlit is missing;
  `_cmd_list_metrics` routes its load through the same clean-error/exit-1 handler.

Purpose: Satisfy CLI-01 (validated arguments + tested error/exit-code paths). Depends on
Plan 01 for `ToolPreflightError` and the adapter `_timeout` field. Plan 03 tests these
paths at the `main()` level.

Output: A restructured `cli.py` with a `rescore` subcommand, a clean-error wrapper, the
corrected exit-code taxonomy, and the `--timeout` flag.
</objective>
