<objective>
Harden the package CLI surface (`cli.py`) so every command validates arguments and exits with
a documented, predictable code, AND so a real sweep/rescore/analyze failure actually produces
a non-zero exit. This plan consumes the typed failure counters and seams Plan 01 produced.

CLI-level routed findings + review HIGH #2:
- **D-09** — promote `rescore` to its own subcommand so the four rescore-only flags
  (`--reuse-parser-options`, `--use-prediction-cache`, `--pass-id`, the rescore meaning of
  `--max-trials`) live on it and argparse structurally rejects them on `run`.
- **D-08 / F-06 / review HIGH #2 + RESOLVED-A** — a minimal POSIX `0` / `1` / `2` exit-code
  taxonomy. **Sweep** (`run`) returns 1 when `summary.trial_failures` or `summary.eval_failures`
  is non-zero, OR when the sweep produced ZERO successful trials (`successful_trials == 0` /
  `best_trial is None`, even if all failures were timeouts — RESOLVED-A) — replacing today's
  unconditional `return 0`. **Rescore** returns 1 on
  `parser_misses` OR `eval_failures`, NOT on pre-existing `skipped_failed`. **Analyze** returns
  1 on `skipped_unreadable` OR `eval_failures`, NOT on benign `skipped_no_case`.
- **D-11 / D-10 / review MEDIUM** — wrap load+dispatch in `main()` in a NARROW clean-error
  handler that catches `FileNotFoundError` / `ValueError` (config loaders) and
  `ToolPreflightError` (the Plan-01 preflight), prints `error: <msg>` with no traceback, and
  returns 1; keep argparse's native exit 2 for usage; let an UNEXPECTED runtime exception keep
  its traceback unless `--traceback` is set (so a programming bug in Ax/metrics is not masked as
  a one-line "config" error).
- **D-04 / review MEDIUM** — add `--timeout` to `run` and apply it via the public
  `set_timeout_override()` seam (not a private `_timeout` write) using `is not None` precedence.
- **Discretion cleanups** — `_cmd_dashboard` returns 1 (not 2) when streamlit is missing;
  `_cmd_list_metrics` routes its load through the same clean-error/exit-1 handler.

Purpose: Satisfy CLI-01 (validated arguments + tested error/exit-code paths) and complete the
0/1/2 taxonomy the review found unmet for sweep failures. Depends on Plan 01 for
`ToolPreflightError`, `set_timeout_override()`, and the typed summary counters. Plan 03 tests
these paths at the `main()` level (including a real sweep-exit-1 test).

Output: A restructured `cli.py` with a `rescore` subcommand, a narrow clean-error wrapper, the
corrected exit-code taxonomy (sweep included), and the `--timeout` flag.
</objective>
