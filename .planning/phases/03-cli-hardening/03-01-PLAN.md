<objective>
Harden the `CliToolAdapter` subprocess contract AND fix the central failure-propagation
defect the Codex review flagged: today a `TrialResult(success=False)` is still parsed,
scored, and reported to Ax as a *completed* trial. This plan lands the adapter-side failure
producers PLUS the runner-side contract that turns them into real Ax failed trials.

Routed findings on the adapter/config surface:
- **F-32 / D-04 / D-05** â€” opt-in `execution.timeout_seconds`; on expiry the subprocess AND
  its descendants (the `conda run` tree) are killed via process-group termination, and the
  trial is recorded as `success=False, error="timeout"`. Unset / `<= 0` = no timeout.
- **F-07 / D-06** â€” `glob` becomes the single blessed output path; an *explicitly* set
  `outputs.from: stdout_json` raises a glob-pointing deprecation error; the default flips to
  `glob`; an unsupported `outputs.from` value (e.g. `fixed_path`) is rejected at load.
- **F-07 / D-07** â€” a set `predictions_glob` matching zero files is a flagged failure
  (`success=False`); an empty `figures_glob` stays non-fatal.
- **F-16 / D-02 / D-10** â€” a fail-fast env/binary preflight in `CliToolAdapter.prepare()`
  raising a structured `ToolPreflightError` (with remediation) before trial 0, robust against
  conda-enumeration failures.

Review HIGH findings #1 + #2 (the heart of this revision):
- **#1** â€” define the runner behavior for `success=False`: skip `evaluate_trial`, and call
  Ax `log_trial_failure` (not `complete_trial`) so a timeout/nonzero/empty-output trial is a
  genuine failed Ax trial and the sweep continues.
- **#2** â€” add typed failure counters to `SweepRunSummary` / `RescoreSummary` /
  `AnalysisSummary` separating tool/runtime + parser/eval failures from benign skips and
  observability degradation, so Plan 02 can base exit 1 on the first two only.

Purpose: Satisfy CLI-02 (robust, propagated subprocess failure handling) and CLI-03 (F2S3
`conda run` preflight) at the code level. Plan 02 consumes the typed counters for exit codes;
Plan 03 tests the runner/Ax propagation and the adapter mechanics; Plan 04 documents them.

Output: A hardened `CliToolAdapter` + loader + `ToolPreflightError` + `TrialExecutionError`
+ the runner failure-propagation contract + typed failure counters on all three summaries.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool.yaml -> CliToolAdapter | operator-supplied entry/args become subprocess argv (trusted config) |
| CliToolAdapter -> OS subprocess (process group) | the tool tree executes with the bench's privileges |
