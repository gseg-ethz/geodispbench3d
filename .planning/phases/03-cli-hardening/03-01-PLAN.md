<objective>
Harden the `CliToolAdapter` subprocess contract and the tool-config loader so subprocess
failures are visible and countable instead of silent. This plan lands four routed audit
findings on the adapter/config surface:

- **F-32 / D-04 / D-05** — opt-in `execution.timeout_seconds` on the tool config; on expiry
  the subprocess is killed and recorded as a counted non-fatal `timeout` failure (sweep
  continues). Unset / `<= 0` means no timeout (today's behavior, now explicit).
- **F-07 / D-06** — `glob` becomes the single blessed output path; an *explicitly* set
  `outputs.from: stdout_json` raises a clear deprecation error pointing at `glob`. The
  default flips from `stdout_json` to `glob`.
- **F-07 / D-07** — a set `predictions_glob` matching zero files is a flagged failure
  (`success=False`, `error="no predictions matched <glob>"`); an empty `figures_glob` stays
  non-fatal.
- **F-16 / D-02 / D-10** — a fail-fast env/binary preflight in `CliToolAdapter.prepare()`
  that raises a structured `ToolPreflightError` with a tool-supplied remediation hint before
  Ax launches trial 0.

Purpose: Satisfy CLI-02 (documented, robust subprocess failure handling) and CLI-03 (F2S3
`conda run` env/binary preflight) at the code level. Plan 02 wires these into the CLI exit
codes; Plan 03 tests them; Plan 04 documents them.

Output: A hardened `CliToolAdapter` + loader + `ToolPreflightError` + the new optional
tool-config fields, with `f2s3.yaml` and the (doc-only) JSON schema updated to match.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool.yaml -> CliToolAdapter | operator-supplied entry/args become subprocess argv (trusted config, not end-user input) |
| CliToolAdapter -> OS subprocess | the tool binary executes with the bench's privileges |
| preflight resolver -> conda CLI | `conda env list --json` is invoked to enumerate envs |

## STRIDE Threat Register (ASVS L1 — block only on `high`)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering/Elevation | entry -> argv | medium | accept | entry is already `shlex.split` with no `shell=True`; argv passed as a list, so no shell-injection surface. Config is trusted operator input. |
| T-03-02 | Elevation | preflight `shutil.which` + binary exec | medium | mitigate | preflight only RESOLVES the leading executable and the named env; it never executes the resolved binary. The tool itself is operator-declared config. |
| T-03-03 | Denial of Service | hung subprocess | high | mitigate | F-32 `subprocess.run(timeout=...)` kills the child and records a counted timeout failure so a hung tool cannot stall the sweep. |
| T-03-04 | Information disclosure | nonzero-exit stdout/stderr logged | low | accept | tool stdout/stderr is logged at error on failure (existing behavior); operator-facing logs, no secret handling in this path. |
| T-03-05 | Spoofing | conda env-name match | low | accept | env membership is checked against the authoritative `conda env list --json`; a deliberately mis-set env name fails the preflight, not silently. |
| T-03-SC | Tampering | npm/pip/cargo installs | n/a | accept | This plan installs no packages — all behavior is stdlib + existing in-repo modules (RESEARCH "Package Legitimacy Audit: Not applicable"). |
</threat_model>
