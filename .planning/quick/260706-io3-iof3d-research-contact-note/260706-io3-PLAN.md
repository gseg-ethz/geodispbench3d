Tell users that the iof3D adapter — which ships dormant until iof3D is public
— is still under active development, and that researchers who want it in the
meantime can contact the maintainer directly at `meyernic@ethz.ch`. The note
must land on `main` (public), and `develop` must not be left behind.

## Decisions (from invocation + discussion)

- **Placement:** README only (augment the existing `[iof3d]` dormancy note).
  Chosen over also editing the runtime `ImportError` hint / package docstring.
- **Target:** land on `main` directly via PR (main is protected). Do NOT
  complete the release-please workflow / cut a tag / publish to PyPI — the
  change is too small to warrant a release.
- **Contact address:** `meyernic@ethz.ch` (already the maintainer email in
  `pyproject.toml`, so no new public exposure).
- **Branch model reality:** `main` is a filtered/squash projection of
  `develop` with no shared SHA lineage (`.planning/` is develop-only). So the
  develop sync is a file-level content sync, NOT a branch merge.
