## Directory Purposes

**`src/geodispbench3d/`:**
- Purpose: The tool-agnostic core framework. Installs with zero tool dependencies.
- Contains: CLI, config loaders, the sweep/rescore/analyze orchestration, persistence, and the dashboard.
- Key files: `cli.py`, `sweep/runner.py`, `sweep/evaluation.py`, `tool/base.py`.

**`src/geodispbench3d/sweep/`:**
- Purpose: Orchestration heart ‚Äî the Ax loop, parameter grammar, evaluation glue, rescore, and per-trial records.
- Contains: `runner.py`, `parameters.py`, `evaluation.py`, `rescore.py`, `trial_record.py`.
- Key files: `runner.py` (`AxSweepRunner`), `evaluation.py` (`evaluate_trial` ‚Äî the shared chokepoint).

**`src/geodispbench3d/tool/`:**
- Purpose: The adapter seam. Everything tool-specific enters here.
- Contains: `base.py` (contract), `cli_adapter.py`, `callable_adapter.py`, `loader.py`.
- Key files: `base.py` (`ToolAdapter`, `TrialRequest/Outputs/Result`), `loader.py` (`load_tool_config`).

**`src/geodispbench3d_iof3d/` and `src/geodispbench3d_f2s3/`:**
- Purpose: First-party tool plugins, each its own top-level package gated by a `pyproject` extra.
- Contains: an output parser, a tool.yaml under `conf/tool/`, and (iof3D only) a factory + in-process adapter + legacy CLI.
- Key files: `geodispbench3d_iof3d/factory.py`, `geodispbench3d_iof3d/adapter.py`, the two `conf/tool/*.yaml`.

**`benchmarks/`:**
- Purpose: The data-side of a run. Split so a dataset/metrics pair can be shared across tools.
- Contains: `data/` (GT), `datasets/`, `metrics/`, `suites/`.
- Key files: `suites/iof3d_mattertal.yaml`, `suites/f2s3_voxel_refine.yaml`.

**`tests/`:**
- Purpose: Install-profile-segmented test suites.
- Contains: `core/` (framework-only), `iof3d/`, `f2s3/`, each with its own `conftest.py`.
- Generated: No. Committed: Yes.

**`src/geodispbench3d/conf/schema/`:**
- Purpose: JSON Schemas (`suite`, `tool`, `dataset`, `metrics`, `analysis`) shipped as package data for external validation.
- Generated: No. Committed: Yes (declared in `pyproject` `package-data`).

## Key File Locations

**Entry Points:**
- `src/geodispbench3d/cli.py`: `geodispbench3d` console script (`main`).
- `src/geodispbench3d_iof3d/cli.py`: legacy `iof3d-ax` Hydra CLI.
- `src/geodispbench3d/dashboard/app.py`: Streamlit dashboard target.

**Configuration:**
- `pyproject.toml`: Single source of build config, dependencies, extras (`dashboard`, `iof3d`, `f2s3`, `dev`, `docs`), ruff, and setuptools_scm.
- `pyrightconfig.json`: Type-checker settings.
- `.pre-commit-config.yaml`: ruff lint/format hooks.
- `src/geodispbench3d/_version.py`: setuptools_scm-generated; never hand-edit (excluded from ruff).

**Core Logic:**
- `src/geodispbench3d/sweep/runner.py`: the Ax trial loop.
- `src/geodispbench3d/sweep/evaluation.py`: parser ‚Üí metric dispatch.
- `src/geodispbench3d/tool/loader.py`: YAML ‚Üí adapter construction.

**Testing:**
- `tests/conftest.py`: shared extras-aware fixtures.
- `tests/core/test_*.py`: framework behavior (loaders, rescore, analyze, predictions cache, metrics).

## Naming Conventions

**Files:**
- `snake_case.py` for all modules.
- Loaders are named `loader.py` (per package) or carry the schema in `schema.py`/`registry.py`.
- Tests mirror the module under test: `test_<module>.py` (e.g. `test_rescore.py`, `test_predictions_cache.py`).
- YAML configs are lowercase, dash-or-underscore separated, named for what they describe (`mattertal.yaml`, `pointing_error.yaml`, `f2s3_voxel_refine.yaml`).

**Directories:**
- Core subpackages are single-word, domain-named (`sweep`, `tool`, `dataset`, `metrics`, `results`, `analysis`, `suite`, `dashboard`).
- Tool plugins use the `geodispbench3d_<tool>` prefix (underscore-suffixed top-level package), gated by a matching `[<tool>]` extra.

**Code symbols:**
- Classes: `PascalCase` (`AxSweepRunner`, `CliToolAdapter`, `ToolProvenance`).
- Functions / variables: `snake_case`.
- Module-private helpers: leading underscore (`_build_adapter`, `_resolve_case`, `_to_jsonable`).
- Public surface of each module declared explicitly via `__all__`.

## Where to Add New Code

**A new tool integration:**
- Create a new top-level package `src/geodispbench3d_<tool>/` with an `output_parser.py` and a `conf/tool/<tool>.yaml`.
- If the tool runs via subprocess, point `tool.yaml` at `kind: cli` and reuse `CliToolAdapter` (no Python adapter needed ‚Äî see F2S3).
- If construction needs more than dataclass kwargs (e.g. building a config object), add a `factory.py` and use `kind: factory` (see iof3D).
- Register the package in `pyproject.toml` `[tool.setuptools] packages`, add a `[<tool>]` extra, and add a `package-data` entry for its `conf/**/*.yaml`.
- Add a self-skipping test dir `tests/<tool>/` with its own `conftest.py`.

**A new metric:**
- Add the function to `src/geodispbench3d/metrics/builtins.py` (kwargs in, JSON-friendly out, declare `needs` via the keyword args it accepts).
- Reference it from a `metrics.yaml` under `benchmarks/metrics/` via `package.module:function`.
- No registry edit required ‚Äî resolution is by dotted path.

**A new benchmark run:**
- Add a `suite.yaml` under `benchmarks/suites/` referencing an existing tool/dataset/metrics trio.
- Add `dataset.yaml`/`metrics.yaml` under `benchmarks/datasets/` and `benchmarks/metrics/` if new.

**A new CLI subcommand:**
- Add a subparser in `src/geodispbench3d/cli.py:main` and a `_cmd_<name>` handler; keep heavy imports inside the handler (lazy-import convention).

**Shared helpers:**
- Cross-mode utilities belong in `src/geodispbench3d/sweep/trial_record.py` (provenance/record I/O) or the relevant domain package ‚Äî there is no catch-all `utils` module, and new code should not introduce one.
